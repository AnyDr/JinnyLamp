# ADR-0001: Wireless архитектура (ESP-NOW + Wi-Fi STA)

## Context
Нужен локальный канал управления (пульт -> лампа) без зависимости от сервера/роутера.
Также нужен путь к будущему Home Assistant через Wi-Fi/MQTT.

## Decision
- ESP-NOW используется для локального управления и обратной связи (ACK + snapshot).
- Pairing по MAC (CONFIG_J_ESPNOW_PEER_MAC) для контролируемой и расширяемой сети.
- Wi-Fi STA поднимается всегда, но:
  - при пустом SSID работает “ESPNOW-only” с фиксированным fallback channel,
  - при заданном SSID подключается к AP и канал диктуется AP (ESPNOW следует каналу STA).
- Источник истины состояния - лампа (ctrl_bus), UI устройства (пульт) - клиент с кэшем.

## Consequences
Плюсы:
- Работает без инфраструктуры (роутер/сервер не обязателен)
- Низкая задержка
- Простая диагностика: команды + ACK со снимком состояния

Минусы/Риски:
- Нужно внимательно управлять Wi-Fi каналом (STA канал vs fallback)
- Требуется синхронизация таблиц effect_id между пультом и лампой
  (или протокол запроса списка эффектов в будущем)
